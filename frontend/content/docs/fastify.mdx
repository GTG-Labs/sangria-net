## 2. Initialize the client

```ts
import { Sangria } from "@sangria-sdk/core";

const sangria = new Sangria({
  apiKey: process.env.SANGRIA_SECRET_KEY,
});
```

## 3. Register the plugin

Fastify requires a one-time plugin registration. This decorates the request object so `request.sangria` is available in your handlers.

```ts
import Fastify from "fastify";
import { sangriaPlugin } from "@sangria-sdk/core/fastify";

const fastify = Fastify();
fastify.register(sangriaPlugin);
```

## 4. Set a price on your endpoint

Use `fixedPrice` as a `preHandler` hook on any route you want to charge for.

```ts
import Fastify from "fastify";
import { Sangria } from "@sangria-sdk/core";
import { sangriaPlugin, fixedPrice } from "@sangria-sdk/core/fastify";

const fastify = Fastify();

const sangria = new Sangria({
  apiKey: process.env.SANGRIA_SECRET_KEY,
});

fastify.register(sangriaPlugin);

// This endpoint is free
fastify.get("/", async () => {
  return { message: "Hello! This endpoint is free." };
});

// This endpoint costs $0.01 per request
fastify.get(
  "/premium",
  {
    preHandler: fixedPrice(sangria, {
      price: 0.01,
      description: "Access premium content",
    }),
  },
  async () => {
    return { message: "You accessed the premium endpoint!" };
  }
);

await fastify.listen({ port: 4002 });
```

### Bypassing payment

If your customers already authenticate with their own API key, you can skip Sangria payment for those requests:

```ts
fastify.get(
  "/premium",
  {
    preHandler: fixedPrice(sangria, {
      price: 0.01,
      description: "Access premium content",
    }, {
      bypassPaymentIf: (request) => {
        const key = request.headers["x-api-key"];
        return typeof key === "string" && isValidApiKey(key); // validate against your own auth system
      },
    }),
  },
  async () => {
    return { message: "You accessed the premium endpoint!" };
  }
);
```

### Accessing payment data

Payment details are available on `request.sangria` inside your handler:

```ts
fastify.get(
  "/premium",
  {
    preHandler: fixedPrice(sangria, { price: 0.01, description: "Access premium content" }),
  },
  async (request) => {
    const payment = request.sangria;
    // payment.paid — true if agent paid
    // payment.amount — the price charged
    // payment.transaction — on-chain tx hash

    return { paid: payment?.paid };
  }
);
```

## Handle errors

If the Sangria backend is *unreachable, times out, or returns a non-2xx status*, the SDK throws a `SangriaError`. Register a Fastify error handler to return a clean response instead of Fastify's default `500`:

```ts
import { SangriaError } from "@sangria-sdk/core";

fastify.setErrorHandler((err, _request, reply) => {
  if (err instanceof SangriaError) {
    fastify.log.error(`[sangria:${err.operation}] ${err.message}`);
    return reply.status(503).send({ error: "Payment provider unavailable" });
  }
  throw err;
});
```

`SangriaError` has an `.operation` field (`"generate"` or `"settle"`) so you know which step failed. Subclasses give you more detail:

| Error | When | Properties |
|-------|------|------------|
| `SangriaConnectionError` | Network failure (DNS, refused, reset) | — |
| `SangriaTimeoutError` | Request timed out | — |
| `SangriaAPIStatusError` | Backend returned 4xx/5xx | `.statusCode`, `.response` |

Business-level payment failures (bad signature, insufficient funds) are **not** errors — they return a normal `402` response to the caller.

## That's it

Your endpoint is now priced. When an AI agent calls it, here's what happens:

1. The SDK returns a `402` with the price and payment details
2. The agent signs a USDC payment off-chain and retries with a `payment-signature` header
3. The SDK settles the payment on-chain, and your handler runs
