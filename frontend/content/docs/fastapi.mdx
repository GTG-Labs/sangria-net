## 2. Initialize the client

```python
import os
from sangria_sdk import SangriaMerchantClient

client = SangriaMerchantClient(
    api_key=os.getenv("SANGRIA_SECRET_KEY"),
)
```

## 3. Set a price on your endpoint

Add the `@require_sangria_payment` decorator to any endpoint you want to charge for.

```python
import os
from fastapi import FastAPI, Request
from sangria_sdk import SangriaMerchantClient
from sangria_sdk.adapters.fastapi import require_sangria_payment

app = FastAPI()

client = SangriaMerchantClient(
    api_key=os.getenv("SANGRIA_SECRET_KEY"),
)

# This endpoint is free
@app.get("/")
async def health():
    return {"message": "Hello! This endpoint is free."}

# This endpoint costs $0.01 per request
@app.get("/premium")
@require_sangria_payment(client, amount=0.01, description="Access premium content")
async def premium(request: Request):
    return {"message": "You accessed the premium endpoint!"}
```

### Bypassing payment

If your customers already authenticate with their own API key, you can skip Sangria payment for those requests:

```python
def has_valid_api_key(request: Request) -> bool:
    key = request.headers.get("x-api-key")
    return key is not None and is_valid_api_key(key)  # validate against your own auth system

@app.get("/premium")
@require_sangria_payment(
    client,
    amount=0.01,
    description="Access premium content",
    bypass_if=has_valid_api_key,
)
async def premium(request: Request):
    return {"message": "You accessed the premium endpoint!"}
```

If your `bypass_if` callback raises, the SDK logs the exception and falls through to the normal payment-required flow — the request is **not** bypassed. This is intentional: a crashing callback cannot silently leak free access to paid endpoints. Write your callback defensively against missing headers, unavailable stores, etc.

### Accessing payment data

Payment details are available on `request.state.sangria_payment` inside your handler:

```python
@app.get("/premium")
@require_sangria_payment(client, amount=0.01, description="Access premium content")
async def premium(request: Request):
    payment = request.state.sangria_payment
    # payment.paid — True if agent paid
    # payment.amount — the price charged
    # payment.transaction — on-chain tx hash

    return {"paid": payment.paid}
```

## Handle errors

If the Sangria backend is *unreachable, times out, or returns a non-2xx status*, the SDK raises a `SangriaError`. Register a FastAPI exception handler to return a clean response instead of a raw `500`:

```python
from fastapi import Request
from fastapi.responses import JSONResponse
from sangria_sdk import SangriaError

@app.exception_handler(SangriaError)
async def sangria_error_handler(_request: Request, exc: SangriaError):
    print(f"[sangria:{exc.operation}] {exc}")
    return JSONResponse(
        status_code=503,
        content={"error": "Payment provider unavailable"},
    )
```

`SangriaError` has an `.operation` attribute (`"generate"` or `"settle"`) so you know which step failed. Subclasses give you more detail:

| Error | When | Properties |
|-------|------|------------|
| `SangriaConnectionError` | Network failure (DNS, refused, reset) | — |
| `SangriaTimeoutError` | Request timed out | — |
| `SangriaAPIStatusError` | Backend returned 4xx/5xx | `.status_code`, `.response` |

Business-level payment failures (bad signature, insufficient funds) are **not** errors — they return a normal `402` response to the caller.

## That's it

Your endpoint is now priced. When an AI agent calls it, here's what happens:

1. The SDK returns a `402` with the price and payment details
2. The agent signs a USDC payment off-chain and retries with a `payment-signature` header
3. The SDK settles the payment on-chain, and your handler runs
