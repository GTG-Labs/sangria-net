## 2. Initialize the client

```ts
import { Sangria } from "@sangria-sdk/core";

const sangria = new Sangria({
  apiKey: process.env.SANGRIA_SECRET_KEY,
});
```

## 3. Set a price on your endpoint

Add `fixedPrice` as middleware to any route you want to charge for.

```ts
import express from "express";
import { Sangria } from "@sangria-sdk/core";
import { fixedPrice } from "@sangria-sdk/core/express";

const app = express();

const sangria = new Sangria({
  apiKey: process.env.SANGRIA_SECRET_KEY,
});

// This endpoint is free
app.get("/", (_req, res) => {
  res.json({ message: "Hello! This endpoint is free." });
});

// This endpoint costs $0.01 per request
app.get(
  "/premium",
  fixedPrice(sangria, {
    price: 0.01,
    description: "Access premium content",
  }),
  (_req, res) => {
    res.json({ message: "You accessed the premium endpoint!" });
  }
);

app.listen(4001);
```

### Bypassing payment

If your customers already authenticate with their own API key, you can skip Sangria payment for those requests:

```ts
app.get(
  "/premium",
  fixedPrice(sangria, {
    price: 0.01,
    description: "Access premium content",
  }, {
    bypassPaymentIf: (req) => {
      const key = req.headers["x-api-key"];
      return typeof key === "string" && isValidApiKey(key); // validate against your own auth system
    },
  }),
  (_req, res) => {
    res.json({ message: "You accessed the premium endpoint!" });
  }
);
```

If your `bypassPaymentIf` callback throws, the SDK logs the error and falls through to the normal payment-required flow — the request is **not** bypassed. This is intentional: a crashing callback cannot silently leak free access to paid endpoints. Write your callback defensively against missing headers, unavailable stores, etc.

### Accessing payment data

Payment details are available on `req.sangria` inside your handler:

```ts
app.get(
  "/premium",
  fixedPrice(sangria, { price: 0.01, description: "Access premium content" }),
  (req, res) => {
    const payment = req.sangria;
    // payment.paid — true if agent paid
    // payment.amount — the price charged
    // payment.transaction — on-chain tx hash

    res.json({ paid: payment?.paid });
  }
);
```

## Handle errors

If the Sangria backend is *unreachable, times out, or returns a non-2xx status*, the SDK throws a `SangriaError`. Without an error handler, Express returns a raw `500` stack trace to the caller.

Add an error handler **after** all your routes:

```ts
import { SangriaError } from "@sangria-sdk/core";

app.use((err, _req, res, next) => {
  if (err instanceof SangriaError) {
    console.error(`[sangria:${err.operation}]`, err.message);
    return res.status(503).json({ error: "Payment provider unavailable" });
  }
  next(err);
});
```

`SangriaError` has an `.operation` field (`"generate"` or `"settle"`) so you know which step failed. Subclasses give you more detail:

| Error | When | Properties |
|-------|------|------------|
| `SangriaConnectionError` | Network failure (DNS, refused, reset) | — |
| `SangriaTimeoutError` | Request timed out | — |
| `SangriaAPIStatusError` | Backend returned 4xx/5xx | `.statusCode`, `.response` |

Business-level payment failures (bad signature, insufficient funds) are **not** errors — they return a normal `402` response to the caller.

## That's it

Your endpoint is now priced. When an AI agent calls it, here's what happens:

1. The SDK returns a `402` with the price and payment details
2. The agent signs a USDC payment off-chain and retries with a `payment-signature` header
3. The SDK settles the payment on-chain, and your handler runs
