## Conventions

- Every backend call from mythos is an admin operation. `mythos/lib/api-proxy.ts` `proxyToBackend` enforces this at the proxy layer via `verifyAdmin()` in addition to the layout-level gate and backend's `RequireAdmin` middleware. If you add a non-admin route in the future, create a separate proxy helper — don't weaken the existing one.
- Every env var read by code in this app must be declared in `lib/env.ts` and validated by its Zod schema (via `@t3-oss/env-nextjs`). Import `{ env }` and reference `env.BACKEND_URL` etc. Never read `process.env` directly elsewhere — new env vars go in `lib/env.ts` schema first, then the `runtimeEnv` mapping (Next.js's build-time inlining requires the literal `process.env.NEXT_PUBLIC_X` reference there). **Narrow exception:** third-party libraries (e.g. WorkOS AuthKit reads `WORKOS_CLIENT_ID`, `WORKOS_API_KEY`, `WORKOS_COOKIE_PASSWORD` from `process.env` internally) cannot be routed through the schema; these are listed in `README.md` § Environment Variables with `Validated: No` for operator visibility. Don't invent additional exceptions — if our code touches the value, it goes in `lib/env.ts`. Buyer-side demo credentials (`SANGRIA_SECRET_KEY`, `BUYER_ADDRESS`, `CDP_API_KEY_NAME`, `CDP_API_KEY_PRIVATE_KEY`, `CDP_WALLET_SECRET`) are read by `app/api/x402-pay/route.ts` and ARE in the schema — they're agent/wallet creds for the demo flow, not the backend's CDP credentials.
- Proxy routes and paginated list patterns: see root CLAUDE.md § Next.js App Conventions.
